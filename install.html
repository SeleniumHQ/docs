Selenium Installation

Selenium set up is quite different than other commercial tools. You don't install a tool using selenium binaries but set up your test automation project to use selenium binary in the langauge of your choice. The selenium binary you would need depends on browsers you want to run your tests upon.

Using browser specific jar

If you want to run tests in only a spcific brower then you can add browser specific dependency in your pom.xml file. You should add following dependency in your pom.xml file to run tests only on Firefox browser - that's all and you are ready to driver your tests in firefox browser.

Using standalone jar

More often than not you would need to run test on more than one browser and you would also like to take advantage of Selenium Grid to distribute tests in a combination of operating systems and browser. Hence you would use selenium-java maven dependency in your project - Adding this dependency would bring following artifacts to your project -
  • selenium-chrome-driver
  • selenium-edge-driver
  • selenium-firefox-driver
  • selenium-ie-driver
  • selenium-safari-driver
Following third party browsers are not artifacts of selenium-java maven dependency and should be configured manually -
  • OperaDriver
  • GhostDriver
  • Appium
  • Selendroid
  • Blackberry
