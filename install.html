Selenium installation

Selenium setup is quite different from setup of other commercial tools. You donâ€™t install a tool using Selenium binaries but rather set up your test automation project to use such binaries in the language of your choice. The Selenium binaries you would need depends on browsers you want to run your tests in.

Using browser specific JAR

If you want to run tests in only a specific browser, then you can add a browser specific dependency in your pom.xml file. You should add the following dependency in your pom.xml file to run tests only in Firefox: 

<dependency>
  <groupId>org.seleniumhq.selenium</groupId>
  <artifactId>selenium-firefox-driver</artifactId>
  <version>2.53.1</version>
</dependency>

That's all and you are ready to drive your tests in Firefox. Similarly, if you want to run tests only in Chrome, then you can use the following dependency: 

<dependency>
  <groupId>org.seleniumhq.selenium</groupId>
  <artifactId>selenium-chrome-driver</artifactId>
  <version>2.53.1</version>
</dependency>

Using non browser specific JAR

More often than not, you would need to run tests in more than one browser. Hence, you would use a selenium-java Maven dependency in your project. The selenium-java JAR contains the language bindings and the drivers but not the server side piece. 

<dependency>
  <groupId>org.seleniumhq.selenium</groupId>
  <artifactId>selenium-java</artifactId>
  <version>2.53.1</version>
</dependency>

Using standalone JAR

If you plan to use Selenium Grid, then you should download the selenium-server-standalone JAR file. This file is never uploaded, but all the components are available via selenium-server. The standalone JAR contains everything, including the remote Selenium server and the client-side bindings. This means that if you use the selenium-server-standalone jar in your project, then you don't have to add selenium-java or a browser specific jar.
