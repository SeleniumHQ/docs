Selenium HTML-runner

Selenium HTML-runner allows you to run Test Suites from a command line. Test Suites are HTML exports from Selenium IDE or campatible - Selenium IDE based tools (like e.g. Monitis Transaction Monitor scenarios generated by Monitis Transaction Recorder browser plugin)

Common information

  • Combination of releases of geckodriver / firefox / selenium-html-runner matters. There might be a software compatibility matrix somewhere.
  • selenium-html-runner runs only Test Suite (not Test Case - what is for example an export from Monitis Transaction Monitor). Be sure you comply with this.
  • For Linux users with none DISPLAY - you need to start html-runner with Virtual display (search for xvfb)

    How to run selenium-html-runner headless

    Now, the most important part. Example of how to run it headless. Your experience might vary depending on software combinations - geckodriver/FF/html-runner releases. You can also grep for scenario result eventually (PASS/FAILED):

    xvfb-run java -Dwebdriver.gecko.driver=/home/mmasek/geckodriver.0.18.0 -jar selenium-html-runner-3.7.1.jar -htmlSuite "firefox" "https://YOUR-BASE-URL" "$(pwd)/testsuite.html" "results.html" ; grep result: -A1 results.html/firefox.results.html
